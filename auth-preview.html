How this route is protected

① middleware.ts — calls supabase.auth.getUser() on every request. No session → redirect to /login?redirectTo=/dashboard
② app/(protected)/layout.tsx — server component re-validates the session before rendering any child page. Defence-in-depth fallback.
③ useAuth() — client-side context keeps session state in sync across the app. Listens to onAuthStateChange.
