# Hero Section Architecture Refactor – Chain (retrieve-your-roots.com)

## Context
The current hero has strong editorial content but suffers from competing CTAs, navigation friction, and inefficient use of above-fold space. The platform's core interaction pattern is **question-driven entry**, but this is buried below the fold.

## Current State Issues
1. Two identical text-link CTAs ("Follow a thread" and "Before slavery") with no visual hierarchy
2. Pre-written questions (6 of them) are positioned below fold but represent the designed onboarding path
3. Navigation (top banner + main nav + search toggle) competes for attention with hero message
4. Mobile responsiveness unclear—stacking order and tap targets need definition
5. No color differentiation between primary/secondary actions

## Desired Hero Architecture

### Reordered Content Stack (Top to Bottom)
```
1. Headline + Subheading
   "The story begins long before 1619."
   "Before the slave ships. Before the plantations..."
   
2. Question-Based Entry Point (NEW POSITION)
   Search box: "What's your question about how history shapes today?"
   + 3 quick-select question pills (rotate from the 6 available)
   
3. Secondary CTA
   Text link: "Or explore African civilizations →"
   
4. Visual Hint (Optional)
   Small icons: 📜 Threads | 🗺 Maps | 📅 Timeline
```

### Visual Hierarchy Changes
- **Primary interaction**: Question search (prominent input field)
- **Secondary interaction**: "Explore" link (text link, subtle)
- **Tertiary**: Nav lives in persistent header, not hero

### Specific Deliverables
1. Remove "Follow a thread" and "Before slavery" as separate CTAs—consolidate into single search-driven entry
2. Move the 6 pre-written questions into quick-select pills below search box
3. Separate nav from hero content visually (header nav should not compete for attention)
4. Test mobile stacking: ensure hero message is readable at 320px without horizontal scroll
5. Define tap target sizes on mobile (min 48x48px for buttons/pills)
6. Add color accent to question search (use brand color for input focus state)
7. Verify WCAG AAA contrast on body copy over background

### Mobile-Specific
- Hero message stays full-width and readable
- Search box is full-width (not constrained)
- Question pills scroll horizontally or stack to 2 columns
- Navigation collapses into hamburger above hero
- Single clear visual entry point (the search field)

### Interaction States to Define
- Search input: default, focus, placeholder text visibility
- Question pills: default, hover, active/selected
- Secondary CTA: default, hover, visited states

## Success Criteria
- Above-fold space shows: headline + search entry point + 1-2 visual hints
- No horizontal scroll on mobile at 320px
- Primary CTA (search) is visually distinct from secondary (explore link)
- New users immediately understand they can search for topics *or* explore freely
- Tap targets meet 48x48px minimum on mobile

## Reference
Design inspiration: Question-driven entry patterns like Quora, Wikipedia search, or StackOverflow—put the user's intent first.
